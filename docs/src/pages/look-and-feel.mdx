# Look & feel

Dogfood's visual system is **Apple / iOS flavored**: SF Pro, white and soft-gray cards,
system colors, and fully-rounded pills — calm, light, and spatial. The app and the docs
both wear it.

## The language: floating capsules on a calm canvas

- A rounded app window with a large **soft shadow**. Chrome is a warm **near-white**
  (off-white, not pure `#fff`), with hairline light-gray borders.
- **Controls are floating rounded capsules** with a soft shadow, sitting on the canvas
  rather than docked to the edges.
- Palette is whisper-quiet: off-whites, light-gray hairlines, **near-black** (`#1c1c1e`)
  for icons and text. System accents used sparingly (blue `#007aff`, green `#34c759`).
- **Active state = solid black rounded square with a white glyph.**
- Thin-stroke, rounded **line icons**. Generous negative space.
- The **canvas is the hero** — a faint grid surface that the preview and terminal float on.

## The terminal is a real terminal

There is **no "Ask Claude" pill** and no special CC mode. Dogfood's terminal is a genuine
shell — *Terminal.app inside Dogfood*. You type `claude` to start Claude Code, just like
in any terminal (and `npm`, `git`, `vim`, anything else).

It's styled to match the system — rounded corners, soft shadow, hairline border, a good
monospace, a calm near-white (or dark) theme — but functionally it is the real thing
(xterm + a real pty).

## On the canvas

The terminal **floats on the canvas** as a free, draggable, resizable window — it and the
preview are independent objects with z-order that can overlap, like windows on a desktop.
The minimized terminal collapses to a small **status pill**; ⌘J summons or focuses it.

(This supersedes the earlier bottom-drawer idea — see decision 14 in
`brainstorming/decisions.md`.)
